# Work Unit: p1-demo-account-login-entry

## Problem
- 포트폴리오 방문자가 계정 생성/승인 절차 없이 빠르게 기능을 체험하기 어려웠다.
- 데모 계정 정보를 항상 노출하면 화면이 번잡해지고 일반 로그인 흐름 집중을 방해할 수 있다.

## Decision
- 로그인 화면에 “데모 계정으로 빠르게 체험하기” 토글 버튼을 추가했다.
- 클릭 시에만 데모 이메일/비밀번호 안내 패널을 펼쳐 보여주는 방식으로 구성했다.

## Trade-offs
- 장점: 데모 진입 장벽을 낮추면서 기본 로그인 UI 가독성을 유지할 수 있다.
- 단점/리스크: 공개 자격증명 노출 구조이므로 계정 데이터 오염을 전제로 운영 안전장치가 필요하다.

## Implementation Summary
- 변경 파일:
  - `apps/web/src/components/AuthLoginPage.tsx`
  - `apps/web/src/App.css`
- 핵심 반영:
  - 로그인 페이지 상태값 `isDemoAccountOpen` 추가.
  - 토글 버튼(`auth-demo-toggle`) + 안내 패널(`auth-demo-panel`) 추가.
  - 데모 계정 안내 문구(공용 계정, 데이터 변동 가능) 반영.
  - 버튼을 눈에 띄는 파란 강조 스타일로 적용.

## How To Test
- 로그인 페이지 진입 시 데모 안내 패널이 기본적으로 닫혀 있는지 확인한다.
- 토글 버튼 클릭 시 패널이 열리고, 다시 클릭하면 닫히는지 확인한다.
- 안내 패널에 데모 이메일/비밀번호가 정확히 표시되는지 확인한다.
- 토글 UI 추가 후 로그인/회원가입 이동 및 기존 인증 플로우가 깨지지 않는지 확인한다.

# k6 후속 테스트 계획

본 문서는 `local-read-matrix-20260512-102647` 결과를 기반으로, 다음 단계의 성능 검증 계획을 정리한 문서입니다.

## 1) 권장 테스트 로드맵

| 테스트 유형 | 목적 | 권장 부하 프로파일 | 핵심 판정 지표 | 기대 산출물 |
|---|---|---|---|---|
| 재현성 검증 (Repeatability) | 수치 변동 폭 확인 | `80 VU, 10m` 동일 조건 3회 반복 | `p95/p99` 편차, RPS 편차, 실패율 | 기준선 분산(최소/중앙/최대) |
| 용량 탐색 (Capacity Step-Up) | 안전 처리 한계 파악 | `20→40→60→...→160 VU`, 각 `5m` 유지 | 단계별 `RPS`, `p95/p99`, 5xx, 401/403 | 성능 저하 시작 지점(브레이크포인트) |
| 스파이크 내성 (Spike) | 급증 트래픽 대응성 확인 | `20 VU 5m → 120 VU 2m → 20 VU 5m` | 급증 시 `p99`, 에러율, 회복 시간 | 충격 시나리오 대응성 |
| 장시간 안정성 (Soak) | 누적 리소스 이슈 탐지 | `80 VU, 2~4h` | 시간 경과에 따른 `p95/p99`, 메모리/GC 추세 | 지연 드리프트/메모리 드리프트 |
| 혼합 워크로드 (Read/Write) | 실제 사용 패턴 근사 | Read:Write=`70:30` 또는 `80:20` | read/write별 p95, lock/경합 지표 | 실사용 유사 성능 |
| 인증 부하 (Auth Churn) | 로그인/토큰 갱신 구간 검증 | 토큰 TTL 단축 + `RELOGIN_ON_401=true` | `/api/auth/login` p95, 401 비율, 재시도율 | 인증 병목/오탐 구분 |
| 데이터 스케일 민감도 | 데이터 증가 영향 측정 | 동일 부하에서 데이터셋 `x1/x10/x50` 비교 | endpoint별 p95/p99 변화율 | 데이터 규모별 성능 곡선 |

## 2) 실행 우선순위

1. `Repeatability`로 기준선 분산 확보
2. `Capacity Step-Up`으로 허용 용량 범위 정의
3. `Spike`로 회복 탄력성 검증
4. `Soak`로 장시간 안정성 검증
5. `Mixed/Auth/Data-scale` 순으로 실서비스 근사도 강화

## 3) 공통 판정 기준 예시

- 에러율: `http_req_failed < 0.1%`
- 응답지연: 주요 read API `p95`/`p99` 허용치 사전 정의
- 안정성: 부하 상승 시 RPS 선형성 붕괴 지점 기록
- 회복성: spike 종료 후 `p95`가 기준 범위로 복귀하는 시간 측정
- 지속성: soak 중 메모리/GC/지연의 단조 증가 패턴 여부 확인

## 4) 실행 결과 기록 권고

- Grafana 동일 시간대 스냅샷 보관

- 동일 프로파일 반복 시 편차 통계(최소/중앙/최대) 누적 관리
